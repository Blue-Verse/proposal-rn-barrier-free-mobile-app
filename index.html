제안 포인트

1. React Native 기반 Android/iOS 동시 출시 직접 수행 경험

유사한 React Native 프로젝트(피트니스 프랜차이즈 통합 앱)에서 Android, iOS, Web 3 플랫폼을 동시에 설계·개발·런칭한 경험을 보유하고 있습니다. 단일 코드베이스 운영과 플랫폼 분기 처리, 빌드/배포 파이프라인 구성 노하우를 즉시 적용할 수 있습니다.

2. 배리어프리 가이드라인 기반 컴포넌트 설계

본 프로젝트의 핵심은 "기능 추가"가 아닌 "접근성 표준 적용"입니다. 기존 화면 기반으로 다음 항목을 체계적으로 적용합니다.

  • 색 대비비 4.5:1 이상 확보 — 디자인 토큰 재정의
  • 최소 터치 타깃 44×44pt — 모든 인터랙티브 요소
  • 스크린리더(TalkBack / VoiceOver) 음성 안내 — accessibilityLabel / Hint / Role 일관 적용
  • 동적 폰트 스케일 — 시스템 폰트 확대 시 레이아웃 깨짐 방지
  • 포커스 순서 명시 — 키보드/스위치 컨트롤 접근성

3. UI/UX 디자인부터 프론트엔드 개발까지 단일 책임

Figma 디자인 원본 작업과 React Native 구현을 동일 작업자가 수행하므로, 디자인-개발 간 핸드오프 손실이 없습니다. 배리어프리 적용 시 발생하는 미세한 간격/색상/폰트 조정 의사결정이 즉시 반영됩니다.

4. 정부지원사업 산출물 표준 대응

정부지원 연구과제 및 인증 심사 대비 산출물 양식(소스코드 일체, 디자인 원본 파일, 테스트 결과 보고서, 접근성 점검 결과)을 표준화하여 인계합니다.

PRD (Product Requirements Document)

본 PRD는 공고에 명시된 과업 범위를 기반으로 구성되었으며, 미팅 시 협의에 따라 상세 내용이 확정됩니다.

1. 프로젝트 배경 및 목적

기 구축·기획된 모바일 앱을 기반으로 '배리어프리(Barrier-Free)' 인증을 획득하기 위한 UI/UX 고도화 및 React Native 프론트엔드 개발 프로젝트입니다.

자사 내부에 백엔드 개발자가 존재하므로, 백엔드 및 관리자 페이지 구축은 과업에서 제외되며 순수 프론트엔드(클라이언트) 영역의 디자인 수정과 화면 개발에 집중합니다.

  • 대상 인증: 모바일 앱 접근성(KS X 1003) 기반 배리어프리 인증
  • 대상 플랫폼: Android, iOS
  • 핵심 기술: React Native
  • 1순위 우선순위: 산출물 완성도

3. 기능 요구사항

3.1 기존 앱 화면 UI/UX 디자인 고도화

  • [필수] 화면별 시각 요소(아이콘, 간격, 컴포넌트 크기) 재배치 및 디자인 수정
  • [필수] 색 대비비(명도 대비) 4.5:1 이상 확보 (일반 텍스트), 3:1 이상 (대형 텍스트)
  • [필수] 최소 터치 타깃 44×44pt 이상 확보
  • [필수] 시스템 폰트 스케일(Dynamic Type / Font Scale) 대응
  • [필수] 아이콘에 텍스트 라벨 또는 대체 텍스트 병행

3.2 배리어프리 기준 React Native 프론트엔드 개발

  • [필수] 모든 인터랙티브 요소에 accessibilityLabel / accessibilityHint / accessibilityRole 적용
  • [필수] 포커스 순서(accessibilityElementsHidden, importantForAccessibility) 명시
  • [필수] 스크린리더 흐름 검증 — TalkBack(Android), VoiceOver(iOS)
  • [필수] 동적 폰트 스케일에 따른 레이아웃 반응형 처리
  • [필수] 로딩/오류 상태에 대한 LiveRegion(accessibilityLiveRegion) 적용
  • [선택] 다크 모드 / 고대비 모드 토글 (협의 시 추가)

3.3 내부 백엔드 연동 지원

  • [필수] 자사 백엔드 API 연동 (인증, 데이터 조회·등록 등)
  • [필수] API 명세 검수 및 응답 데이터 매핑
  • [필수] 네트워크 오류 시 접근성 알림 및 재시도 처리

5. 기술 제약사항

  • 기존 앱 화면 구성 및 비즈니스 로직 유지 — 접근성 향상 목적 외 기능 변경 없음
  • 자사 백엔드 API 명세에 종속 — 명세 변경 시 일정 협의 필요
  • React Native 기반 단일 코드베이스로 Android·iOS 동시 대응
  • 인증 심사 대비 산출물 표준화 — 디자인 원본, 소스 코드, 점검 결과 포함
  • 네이티브 모듈 신규 개발 필요 시 별도 협의 (현재 범위 외)

6. 범위 외 사항 (Out of Scope)

  • 백엔드 서버 및 API 신규 개발 (의뢰사 자체 수행)
